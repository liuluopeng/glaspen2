# Plan: Fix glaspen2 Windows Pen Input Architecture

## Problem Summary

The app is a transparent overlay for pen drawing. On macOS it works perfectly. On Windows, 4 issues:
1. **Resolution changes** at runtime → Fix: SetProcessDpiAwareness
2. **Strokes invisible during drawing** → Fix: proper rendering pipeline
3. **Mouse becomes hourglass** → Fix: proper message forwarding
4. **Mouse can't operate underlying text** → Fix: proper message forwarding

## Root Cause

**The user's pen sets `dwExtraInfo=0x0` on legacy mouse messages.** `WH_MOUSE_LL` CANNOT distinguish pen from mouse. The ONLY way to receive pen input is WM_POINTER, which requires a non-layered, hit-testable window.

## Solution: Fullscreen non-layered window with proper forwarding

### Files to Modify

- `src/windows/overlay.rs` — window creation, wnd_proc with forwarding
- `src/windows/input.rs` — simplify to direct pixel drawing only
- `src/windows/mod.rs` — DPI awareness

### Implementation

1. **Window**: `WS_EX_TOPMOST | WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE`, DWM transparent, NOT layered
2. **Rendering**: WM_PAINT + AlphaBlend from DIB
3. **Pen input**: WM_POINTER handled in wnd_proc
4. **Message forwarding**: All non-pointer messages forwarded via PostMessageW to window below
5. **Hook**: Only swallow events with actual pen ExtraInfo (is_pen=true), pass everything else through
6. **DPI**: SetProcessDpiAwareness at startup
